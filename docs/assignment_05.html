Instructions

Your starting point for this assignment is the django tutorial you went through for assignment 4. In this assignment, we are going to update our web app so that users can add and remove questions and choices themselves (rather than going through the admin interface).

Change your web app such that:

  1. Each question has a "remove" button after it. Clicking that button will remove the question from the database and refresh the page (refreshing the page can be accomplished with a RedirectResponse)
  2. The page where your questions are listed should have a text field with a button. Clicking the button will submit the form, adding a question with the text from the text field.
  3. Repeat this process for choices. NOTE: choices will be slightly more complex, because each choice must be associated with a question.
  4. Lastly, you will write a brief description of a website that you would like to make for assignment 6. You MAY work with other students on this part. Put your description of the website and the names of any collaborators in a file called proposal.txt in your assignment submission on Github.
  5. Your website should include some ability for users to interact with it (in other words, it should not be a static site.)
  6. Your website must be built with Django

The length of your proposal should be relatively short—how would you describe what your website does to a friend that you are trying to get to use it? Just the name of the website is insufficient.

As always, feel free to ask questions in Discord if you get stuck or stumped.
